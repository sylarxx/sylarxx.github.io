Git总结

  • Git为分布式版本控制系统（本地有完整的版本库，通过联网交换修改）。
  • SVN、CVS为集中式的版本控制系统（必须联网，版本库是集中存放在中央服务器中）。

版本库 repository

一个仓库，只能跟踪文本文件的改动。

git init

将当前目录变成Git可以管理的仓库，该目录下将多一个.git的目录，用来跟踪管理版本库。

首次需要绑定你的邮箱和你的用户名。

git add <文件名>

  • 将文件添加到暂存区（stage）
  • 一次可添加多个文件，git add <文件1> <文件2> <文件3>

git commit -m “备注”

将文件提交到仓库。

git status

显示仓库的状态。

git log

查看提交到仓库里的历史记录/日志。
git log —pretty=oneline 简洁版

git reflog

记录你的每一次命令，可找回你已经删除的版本号（commit id）

工作区、暂存区、版本库

撤销：

  • 工作区：git checkout —<文件名>
  • 暂存区：git reset HEAD <文件名>
  • 分支：git reset —hard <commit id>/HEAD^（返回上个版本）（返回上上个版本 可用HEAD^^，以此类推）

差异：

  • 工作区与暂存区差异：git diff <文件名>
  • 暂存区与分支差异：git diff —cashed <文件名>
  • 工作区与分支差异：git diff HEAD —<文件名>

删除文件

  • 工作区删除：rm <文件名>
  • 版本库删除：git rm <文件名> （需重新commit）

git push origin <分支名>

  • 本地将分支名提交远程库（origin）
  • git push -u origin 分支名 第一次推送分支需要加-u
  • 若失败，git pull抓取最新版本

git remote add origin <git@server-name：path/repo-name.git>

关联一个远程库。
eg：git remote add origin git@github.com:sylarxx/hello-world.git）

git clone <git@server-name：path/repo-name.git>

从远程库克隆到本地。

git branch <分支名>

创建新分支。

git checkout <分支名>

切换到该分支。（git checkout -b <分支名>创建并切换到新分支）

git branch

查看当前分支。

git merge <分支名>

合并指定分支到当前分支。

git branch -d <分支名>

删除该分支。（git branch -D <分支名>删除未合并过的分支）

git stash & git stash pop

  • git stash可将做到一半的工作储藏
  • git stash pop回到之前的工作

git remote -v

查看远程库信息。

git checkout -b <分支名> origin/<分支名>

在本地创建和远程对应的分支。

git branch —set-upstream <分支名> origin/<分支名>

将本地与远程分支关联。

标签相关命令

  1. 创建标签：git tag <tag-name> <commit id>/HEAD
  2. 给标签信息：git tag -a <tag-name> -m “备注”
  3. 查看标签信息：git tag
  4. 删除标签：git tag -d <tag-name>
  5. 推送一个本地标签：git push origin <tag-name>
  6. 推送全部未推送的标签：git push origin —tags
  7. 删除一个远程标签：git push origin :refs/tags/<tag-name>
